Traditional network security relied on perimeter defenses—the "castle-and-moat" model. Zero Trust Architecture (ZTA) updates this by operating on the core assumption that threats exist both outside and inside the network boundary.

## 1. The Core Principles of Zero Trust

Zero Trust is defined by three fundamental tenets:

1. **Verify Explicitly:** Always authenticate and authorize based on all available data points, including user identity, location, device health, and service context.
2. **Use Least Privileged Access:** Limit user access with Just-In-Time and Just-Enough-Access (JIT/JEA) parameters, protecting data and productivity.
3. **Assume Breach:** Minimize blast radius by segmenting networks, encrypting end-to-end sessions, and using analytics to detect anomalies.
