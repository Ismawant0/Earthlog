Extended Berkeley Packet Filter (eBPF) is a revolutionary technology that allows developers to run sandboxed programs inside the Linux kernel without changing kernel source code or loading external modules.

## 1. What is eBPF?

Historically, adding observability or networking capabilities to the Linux kernel required loading kernel modules, which carried the risk of crashing the entire system. eBPF solves this by providing a safe, verified virtual machine inside the kernel.

## 2. Kernel Hookpoints

eBPF programs are event-driven and are triggered when the kernel passes specific hookpoints:

- **kprobes / kretprobes:** Attach to arbitrary kernel function entries and exits.
- **uprobes / uretprobes:** Attach to user-space applications and libraries dynamically.
- **tracepoints:** Static trace hooks compiled directly into the kernel.
- **XDP (eXpress Data Path):** High-performance packet processing directly in the network driver.
