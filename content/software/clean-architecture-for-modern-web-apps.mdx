In modern web development, frameworks and platforms change rapidly. Clean Architecture (often described alongside Hexagonal or Ports-and-Adapters Architecture) protects business rules from dependencies on libraries, database clients, or visual renderers.

## 1. Dependency Rule

The core design principle of Clean Architecture is the Dependency Rule: code dependencies must only point inwards, towards the core business domain.

- **Entities (Domain Core):** Encapsulate enterprise-wide business rules.
- **Use Cases:** Define application-specific operations.
- **Controllers & Gateways (Adapters):** Translate external API payloads into core entities.
- **Web & DB (External Layer):** Frameworks, database clients, and UI renderers.
